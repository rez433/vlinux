länk till videon på terminalen: asciinema record från terminalen

1) Är du sedan tidigare bekant med Unix, Linux, Debian och/eller terminalen?
lite, använder pop os (debian distro) och har använt terminalen för underhålla datorn.

2) Hur känns det med Unix-kommandon på terminalen, är det udda eller bekvämt?
Det är bekvämt och förutom kurs-material som är väldig bra, finns gott om information på nätet. så man känner sig trygg att man hittar all information som behövs för att lära sig mer.

3) Valde du att köra standard med VirtualBox och Debian eller hur gjorde du?
yes, installerade enligt instruktionen förutom ram-minnet som jag tilldelat 2gb. Och språket som jag ville ha på engelska. Använde "dpkg-reconfigure locales" för att ändra på språket.

4) Hur kändes det att jobba med VirtualBox och ser du fördelar med det arbetssättet?
Virtualbox känns samma som vanlig dator med os. Det som är fördelar är att man kan testa och köra olika os utan risk att installera om sin dator och programvaror. Allt händer i virtualbox och man kan ha många virtual machines och testa och köra vilket program som helst. Och Debin fungerar väldig smidig i VirtualBox.

5) ur gick det att installera Guest Additions?
Det gick smidig utan krångel. Allt gick enligt instruktionen.

6) Var det något som krånglade eller tog extra mycket tid?
Inget som krånglat.

Kmom03

1) Är detta din första bekantskap med skriptprogrammering i Bash?
ja, Vi har skrivit commands i terminalen tidigare men första gången att skapa skript-filen.

2) Berätta om din uppfattning om Bash som programmeringsmiljö, relatera till andra programspråk du kan.
Bash språket har liknande struktur som andra c-familj språken. T.ex. while-loop, for in och for-loop är nästan samma som andra programmeringspråk. T.ex. man skriver specefik "do" och "done" istället för curly-braces i Javascript. Man kan skicka parametrar till function och den har extra flexibilitet att de parametrar kan tas emot via varjablar liksom $1, $2 ... 

                
                    function app-loop
                    {
                        for (( i=$1; i<=$2; i++))
                        do
                            echo "$i"
                        done
                    }

3) Vilka möjligheter/utmaningar ser du med denna typen av skriptprogrammering?
möjligheterna är många liksom att automatisera många funktioner i en os. Beroende på ens kunskap i terminalen kan denna person skriva och köra de funktioner i en och samma skript fil. Liksom vi har använt bash-skript i docker för att uppdatera os och sedan installer några package och till och med köra några kommandon. Utmaningar är att man behöver lära sig mycket om linux som os för att skriva ett skript och debugga det.

4) Var det något som var extra svårt eller utmanande i uppgifterna? Har du arbetat med Docker innan?
Att skriva skript-filen var lite klurig eftersom jag använde example skriptet från instruktionen som bas för mitt skript och kunde inte ta emot arguemnt i funktionen. Det tog mig lite tid att förstå problemet och debugga det. I bas skriptet skickas argument som string och inte som list. ändrade "$#" till "$@". Jag har inte jobbat med docker tidigare men det är bra teknologi.

5) Anser du att det är någon fördel/nackdel med Docker om du jämför med VirtualBox?
Som vi läste i instruktionen finns det flera fördelar med docker, att den är lite och använder systemet resurser istället för virtualbox som skapar en hel os. Docker container är lite och kan delas ut och använas enkelt.
